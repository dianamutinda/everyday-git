What is git

Git is a popular version control system.It helps track revisions, collaborate with others, and maintain a history of project development.

Other version control systems

  • Fossil: A distributed VCS, bug tracking system, and wiki software server.
  • Apache Subversion (SVN): A centralized VCS that stores changes as deltas to files.
  • Bazaar: A decentralized VCS that emphasizes simplicity and ease of use.

Remember, in this blog, we will focus on Git and GitHub.

Installing Git

Your operating system does not have git installed by the manufacturers.To check whether git is installed use your computer command prompt in windows or terminal in linux or macos. Type the command git --version if an error message is displayed, download git from here.If the operating system has git git version yyy will be displayed.

Configuring Git

After installing git, its important to let git know who you are by inputing your username and email address. Note that its advisable to use your github user name and email address.

Intializing a Git repository

Before we can use git in our project, we need to initialize a git repository inside the project.To confirm whether the project contains a git repository, we use git status command.You will recieve fatal not git repository... if the project does not contain a git repository, and otherwise if the project contains a git repository.

If the project does not contain a git repository, use git init command to initialize a git repository in the project.

Adding files to the staging area

Use the following command to stage your changes git add.

Tracked Files

Tracked files are those that Git knows about and actively manages. They have been added to the Git repository and have a version history.

Untracked Files

Untracked files are everything else in your working directory.Files that were not in your last commit.Files that are not currently staged.Untracked files don't have a version history in Git.

Commiting changes

Committing Changes in Git

Stage Your Changes by ensuring that you've made the necessary changes to your files in the working directory.Create a Commit once your changes are staged, using the following command git commit -m "Your meaningful commit message here"

Writing Good Commit Messages

  • Be Descriptive:Explain what the commit does. Avoid vague messages like “Fix bug” or “Update code.”
  • Keep It Short:Aim for a concise summary (usually one line) in the commit message.

To see a list of past commits, use git log command.

Branches

In Git, a branch is a lightweight movable pointer to a specific commit. It represents an independent line of development within a Git repository.Each branch has its own commit history, allowing you to work on different features, bug fixes, or experiments simultaneously without affect

Creating a branch

To create a branc you use git branch branch-name command.

Viewing all branches

To view all branches in a repository, use git branch command. There is an aesterik (*) sign in the active branch.

Switching active branches

To switch active branch use the command git checkout branch-name.

Merging a branch

To merge a git branch use git merge branch-name command.

Deleting a branch

To delete a branch in the repository,use the command git branch -d branch-name.
